# Adaptive Rule Refinement using LLMs and Counterfactual Explanations

## Overview

This project implements an adaptive rule refinement pipeline for safety-critical decision systems.

Starting from an imperfect symbolic safety rule, the system:

1. Detects inconsistencies between rule predictions and ground-truth labels.
2. Generates counterfactual explanations for unsafe decisions.
3. Mines recurring unsafe patterns.
4. Uses an LLM to propose refined symbolic rules.
5. Sanitizes and validates generated rules.
6. Accepts only rules that improve safety performance.

The goal is to combine:

* Symbolic reasoning
* Counterfactual explanations
* Large Language Models
* Rule validation

into a reliable adaptive refinement framework.

## Features

### Synthetic Safety Dataset

Generated driving-related scenarios containing:

* speed
* distance
* lane_offset
* weather_risk
* road_curvature

Labels:

* Pass
* Fail

---

### Counterfactual Explanations

For each inconsistency:

```text
Rule predicts PASS
Ground truth is FAIL
```

the system generates a minimal change that would make the situation safe.

Example:

```text
Original:
speed = 19.4
distance = 3.44
label = Fail

Counterfactual:
speed = 19.4
distance = 5.19
label = Pass
```

---

### Pattern Mining

Unsafe regions are mined automatically from counterfactual evidence.

Example:

```text
speed > 14 AND distance < 5

weather_risk > 0.6 AND distance < 6
```

---

### LLM-based Rule Refinement

The LLM receives:

* Original rule
* Unsafe patterns
* Counterfactual evidence

and proposes a refined symbolic rule.

Example:

```text
distance >= 2.2
AND lane_offset <= 1.5
AND weather_risk <= 0.7
AND NOT (speed > 14 AND distance < 5)
AND NOT (weather_risk > 0.6 AND distance < 6)
```

---

### Rule Sanitization

LLM outputs are normalized before validation.

Supported repairs:

* Removing markdown
* Removing explanations
* Normalizing spacing
* Converting:

```text
NOT(A OR B)
```

into:

```text
NOT(A) AND NOT(B)
```

---

### Rule Validation

Each refined rule is evaluated against the full dataset.

Metrics:

* decided cases
* mismatches
* invalid evaluations
* accuracy

Only valid rules are accepted.

## Installation

Create a virtual environment:

```bash
python -m venv venv
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

---

## Configuration

Edit:

```text
config/config.json
```

Example:

```json
{
  "ollama": {
    "max_iterations": 5
  }
}
```

## Usage

Generate dataset:

```bash
python src/dataset_generator.py
```

Run the adaptive refinement pipeline:

```bash
python src/main.py --config config/config.json
```

Visualize results:

```bash
python src/visualize_results.py
```
